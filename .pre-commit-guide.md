# Pre-Commit Hooks Quick Reference

## First-Time Setup

```bash
make dev
```

This installs dependencies and configures pre-commit hooks.

## What Happens on Commit?

When you run `git commit`, these checks run automatically:

1. **File Formatting**: Removes trailing whitespace, ensures newlines
2. **Security**: Detects private keys, blocks large files (>500KB)
3. **Validation**: Checks YAML, TOML, JSON syntax
4. **Ruff**: Lints and formats Python code
5. **Mypy**: Type checks Python code in `src/`
6. **Python Checks**: Prevents `eval()`, requires specific noqa codes

If any check fails, your commit is blocked. Fix the issues and try again.

## Common Scenarios

### Auto-fixable issues
Many issues are fixed automatically (formatting, trailing whitespace, etc.). Just stage the changes and commit again:

```bash
git add .
git commit -m "Your message"
```

### Linting errors
Ruff will show errors that need manual fixing:

```bash
# View the error
# Fix the code
git add .
git commit -m "Your message"
```

### Type errors
Mypy requires type annotations:

```python
# Bad
def process_data(data):
    return data

# Good
def process_data(data: dict[str, Any]) -> dict[str, Any]:
    return data
```

## Manual Testing

Run hooks without committing:

```bash
# All hooks on all files
pre-commit run --all-files

# Specific hook
pre-commit run ruff --all-files
pre-commit run mypy --all-files
```

## Emergency Override

If you absolutely must commit without running hooks (NOT RECOMMENDED):

```bash
git commit -m "Your message" --no-verify
```

Document in your commit message why you skipped hooks.

## Updating Hooks

Update to latest versions:

```bash
pre-commit autoupdate
```

## Alignment with Make Commands

Pre-commit hooks complement existing make targets:

- `make lint` - Same as ruff hook, runs on all files
- `make format` - Same as ruff-format hook, runs on all files
- `make type-check` - Same as mypy hook, runs on src/
- `make check` - Runs lint + type-check + tests

Use make commands for comprehensive checks before pushing.

## Performance

Hooks typically add 2-5 seconds to commit time. They only run on staged files, not your entire codebase.
