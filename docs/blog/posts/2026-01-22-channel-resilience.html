Self-Healing Channels

Networks fail. Connections drop. Bots timeout. That's life.

Today, Klabautermann learned to pick itself up.

The Problem

Telegram's API has hiccups. Your home network goes down for a minute. The webhook times out. The channel marks itself as ERROR and stays there.

You come back an hour later: "Why isn't my Telegram bot responding?"

Because nothing tried to restart it.

The Solution

Auto-restart with exponential backoff:

config = ChannelConfig(
    auto_restart=True,           # Try to recover
    max_restart_attempts=3,      # Give up after 3 tries
    restart_backoff_seconds=5.0, # Start with 5s delay
)

First failure: wait 5 seconds, restart. Second failure: wait 10 seconds, restart. Third failure: wait 20 seconds, restart. Fourth failure? Give up and notify.

The Health Check

The ChannelManager already ran periodic health checks. Now it does something about unhealthy channels:

async def _perform_health_checks(self, stale_threshold: float) -> None:
    for name, channel in self._channels.items():
        # ... existing health check logic ...

        if not is_healthy:
            if self._config.auto_restart:
                await self._attempt_auto_restart(name, error)

Unhealthy? Attempt restart. Within limits.

Tracking Attempts

Each channel tracks its restart attempts:

manager.get_restart_attempts("telegram")  # How many times?
manager.reset_restart_attempts("telegram") # Clear the counter

Successful restart? Counter resets to zero. The channel earned back its trust.

Failure Callbacks

Sometimes you need to know when things break:

def notify_failure(channel_name: str, error: str) -> None:
    send_slack_alert(f"Channel {channel_name} failed: {error}")

manager.on_failure(notify_failure)

Async callbacks work too:

async def async_notify(channel_name: str, error: str) -> None:
    await send_webhook(channel_name, error)

manager.on_failure(async_notify)

Broadcast

Second feature: cross-channel messaging. Send to everyone at once:

result = await manager.broadcast(
    "System maintenance in 10 minutes",
    exclude_channels=["test"],  # Skip the test channel
)

print(f"Delivered to {result.delivered_count} channels")
print(f"Failed: {result.channels_failed}")

Each channel receives the message on its own thread:

result = await manager.broadcast(
    "Important update",
    thread_ids={
        "cli": "system-announcements",
        "telegram": "telegram-123456789",
    },
)

The BroadcastResult

Full transparency on what happened:

@dataclass
class BroadcastResult:
    timestamp: datetime
    content: str
    results: dict[str, bool]    # channel -> success
    errors: dict[str, str]      # channel -> error message
    delivered_count: int
    failed_count: int

    @property
    def all_delivered(self) -> bool: ...
    @property
    def channels_delivered(self) -> list[str]: ...
    @property
    def channels_failed(self) -> list[str]: ...

Know exactly what went where.

Targeted Messages

Sometimes you need to reach one specific channel:

success = await manager.send_to_channel(
    channel_name="telegram",
    content="Just for Telegram users",
    thread_id="telegram-123456789",
)

if not success:
    logger.warning("Telegram message failed")

Returns False if the channel doesn't exist or isn't running.

Configuration

Environment variables for everything:

CHANNEL_AUTO_RESTART=true
CHANNEL_MAX_RESTART_ATTEMPTS=3
CHANNEL_RESTART_BACKOFF=5.0

Or programmatic:

config = ChannelConfig(
    auto_restart=True,
    max_restart_attempts=5,
    restart_backoff_seconds=10.0,
)
