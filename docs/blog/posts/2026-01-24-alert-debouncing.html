Silencing the Echo

The Officer of the Watch monitors your deadlines and meetings. But nobody wants the same warning repeated every 15 minutes.

Alert debouncing solves this.

The Problem

The Officer runs periodic checks. If your task is due in 10 hours, it generates a deadline warning. If nothing changes, the next check generates the same warning again.

Alert fatigue is real. Too many alerts means you ignore all alerts.

The Solution

Track what was sent and when. Before sending an alert, check if the same alert was sent recently:

def _recently_sent(self, alert: Alert) -> bool:
    """Check if a similar alert was recently sent."""
    key = f"{alert.alert_type.value}:{alert.entity_uuid or 'none'}"
    last_sent = self._recent_alerts.get(key)

    if last_sent is None:
        return False

    debounce_seconds = self.officer_config.alert_debounce_minutes * 60
    elapsed = (datetime.now() - last_sent).total_seconds()

    return elapsed < debounce_seconds

The Key

Debounce key = alert_type:entity_uuid

  • deadline_warning:task-123 - Deadline warning for task 123
  • meeting_reminder:event-456 - Meeting reminder for event 456
  • anomaly:none - System-wide anomaly (no entity)

This means:

  • Same alert type + same entity = debounced
  • Different alert type + same entity = not debounced
  • Same alert type + different entity = not debounced

Configuration

The window is configurable:

@dataclass
class OfficerConfig:
    # ...
    alert_debounce_minutes: int = 60  # Default: 1 hour

Set it to 30 for half-hour cooldown. Set it to 0 to disable debouncing entirely (not recommended).

Integration

The debouncing happens automatically in check_conditions():

for alert in all_alerts:
    if self._should_notify(alert):
        alerts_to_send.append(alert)
        self._mark_alert_sent(alert)

_should_notify() checks both quiet mode (Deep Work) and debouncing. Only alerts that pass both filters get sent.

Testing

  • Test different alert types aren't cross-debounced
