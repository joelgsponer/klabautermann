The Art of the TLDR

Skills are the vocabulary of Klabautermann's capabilities. Each one defines what the system can do, when it should do it, and how it integrates with the orchestration layer. Today we added a skill that ties together yesterday's work on email thread summarization with the existing conversation summarization infrastructure.

What Is a Skill?

In Klabautermann's architecture, a skill is a YAML-fronted markdown file that defines:

  • What it does - The name and description
  • When to use it - Trigger phrases for AI-first discovery
  • How it integrates - Which agent handles it, what payload it expects
  • Instructions - Detailed execution steps for the agent

Skills live in .claude/skills/ and are loaded at startup. The orchestrator uses AI-first discovery to match user requests to skills based on semantic understanding, not keyword patterns.

The Skill Definition

Here's the complete summarize-thread skill:

---
name: summarize-thread
description: Summarize a conversation thread or email thread to extract key points,
  action items, and decisions. Use when user asks "summarize this thread",
  "what was discussed", "give me the gist", or "TLDR".
allowed-tools: Read, Grep
model: claude-3-5-haiku-20241022
user-invocable: true
klabautermann-task-type: research
klabautermann-agent: researcher
klabautermann-blocking: true
klabautermann-payload-schema:
  thread_id:
    type: string
    required: false
    extract-from: user-message
    description: Thread ID for email or conversation thread (if known)
  thread_type:
    type: string
    required: false
    default: conversation
    extract-from: user-message
    description: Type of thread - "email" or "conversation"
  query:
    type: string
    required: false
    extract-from: user-message
    description: Search query to find the thread if ID not provided
---

Anatomy of the Frontmatter

Let's break down the key sections:

Standard Claude Code Fields

name: summarize-thread
description: Summarize a conversation thread...
allowed-tools: Read, Grep
model: claude-3-5-haiku-20241022
user-invocable: true

These are the standard fields that Claude Code recognizes. The description is crucial - it's what the AI uses to match user requests like "TLDR" or "what was discussed" to this skill.

Klabautermann Integration

klabautermann-task-type: research
klabautermann-agent: researcher
klabautermann-blocking: true

These fields tell the orchestrator how to handle the skill:

  • task-type: research - This is an information retrieval task
  • agent: researcher - Route to the Researcher agent
  • blocking: true - Wait for the result before responding

Payload Schema

klabautermann-payload-schema:
  thread_id:
    type: string
    required: false
    extract-from: user-message
  thread_type:
    type: string
    default: conversation
  query:
    type: string
    extract-from: user-message

The payload schema defines what information to extract from the user's message. The LLM extracts these fields automatically:

  • thread_id - If the user mentions a specific thread ID
  • thread_type - "email" or "conversation" (defaults to conversation)
  • query - Search terms to find the thread ("from Sarah", "project meeting")

The Discovery Flow

When a user says "TLDR on that email from Sarah", here's what happens:

  1. Skill Discovery: The LLM matches "TLDR" to our skill's description
  2. Payload Extraction: The LLM extracts thread_type: "email", query: "Sarah"
  3. Task Planning: Orchestrator creates a research task for the Researcher agent
  4. Execution: Researcher finds the email thread and summarizes it
  5. Response: Summary returned to the user

The Skill Body

Below the frontmatter, the markdown body contains detailed instructions:

# Summarize Thread

Summarize a conversation thread or email thread to extract structured information.

## Instructions

1. **Determine Thread Type**
   - If thread_type is "email" or user mentions emails/inbox, use email thread summarization
   - Otherwise, use conversation thread summarization

2. **Find the Thread**
   - If thread_id is provided, use it directly
   - If query is provided, search for matching threads
   - If neither, summarize the current/recent conversation

3. **Generate Summary**
   - Extract the essence, not verbatim transcription
   - Identify key topics discussed
   - List decisions made
   - Extract action items with owners if mentioned
   - Detect sentiment (positive/neutral/negative)

These instructions guide the agent when executing the skill. They provide the context the LLM needs to handle edge cases and produce consistent output.

Connecting the Pieces

This skill ties together multiple capabilities:

  • Email Thread Summarization (Issue #221): summarize_email_thread() in GoogleWorkspaceBridge
  • Conversation Summarization: summarize_thread() in the Archivist's summarization module
  • Skill Discovery: AI-first matching via SkillDiscovery
  • Orchestration: Task routing via SkillAwarePlanner

The skill doesn't implement summarization itself - it's a declarative interface that routes user requests to the appropriate implementation.

Fixing .gitignore

One small gotcha: the .claude/ directory was being ignored, which blocked the skills directory. The fix:

# Before (broken):
.claude/*
!.claude/skills/
.claude/          # This line overrode the exception

# After (working):
.claude/*
!.claude/skills/
!.claude/skills/**

The double-star pattern ensures nested directories under .claude/skills/ are also included.
