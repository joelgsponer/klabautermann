Right Model, Right Task: Multi-Model Orchestration

Not every task needs a battleship. Some just need a dinghy.

Previously: Klabautermann used Sonnet for everything. Now: dynamic model selection picks the right tool for each task.

The Problem

Claude models come in different sizes. Haiku is fast and cheap. Sonnet is balanced. Opus is the most capable but slowest and most expensive.

Using Opus for "classify this intent" is like using a sledgehammer to hang a picture frame. Using Haiku for "plan this complex multi-step task" is like bringing a spoon to a sword fight.

We were using Sonnet for everything. Good, but not optimal.

The Solution: ModelSelector

The new ModelSelector picks the appropriate model based on task purpose or complexity:

from klabautermann.core import (
    ModelSelector,
    TaskPurpose,
    TaskComplexity,
    ModelSelectionConfig,
)

selector = ModelSelector(anthropic_client)

# Select by purpose
result = await selector.call(
    prompt="Is this a search, action, or ingestion?",
    purpose=TaskPurpose.CLASSIFICATION,  # -> Haiku
)

# Select by complexity
result = await selector.call(
    prompt="Plan the implementation of this feature...",
    complexity=TaskComplexity.COMPLEX,  # -> Opus
)

Task Purpose Mapping

Each purpose maps to a default model tier:

Purpose Model Why
CLASSIFICATION Haiku Fast pattern matching
EXTRACTION Haiku Structured data extraction
SEARCH_PLANNING Haiku Query construction
REASONING Sonnet Balanced reasoning
SYNTHESIS Sonnet Response generation
ACTION Sonnet Tool execution reasoning
PLANNING Opus Complex multi-step reasoning

Complexity-Based Selection

Don't know the purpose? Select by complexity:

class TaskComplexity(str, Enum):
    SIMPLE = "simple"      # -> Haiku
    MODERATE = "moderate"  # -> Sonnet
    COMPLEX = "complex"    # -> Opus

Configuration Override

Defaults not right for your use case? Override them:

config = ModelSelectionConfig(
    # Custom model IDs
    models={
        ModelTier.HAIKU: "claude-3-5-haiku-latest",
        ModelTier.SONNET: "claude-sonnet-4-latest",
        ModelTier.OPUS: "claude-opus-4-latest",
    },

    # Upgrade extraction to Sonnet for better accuracy
    purpose_overrides={
        TaskPurpose.EXTRACTION: ModelTier.SONNET,
    },

    # Always use Opus for the researcher
    agent_overrides={
        "researcher": ModelTier.OPUS,
    },

    # Fall back to Sonnet if primary fails
    fallback_tier=ModelTier.SONNET,
)

selector = ModelSelector(client, config)

Automatic Fallback

If the primary model fails (rate limit, timeout, error), the selector automatically tries the fallback:

result = await selector.call(
    prompt="Complex planning task...",
    purpose=TaskPurpose.PLANNING,  # Primary: Opus
)

# If Opus fails, result.used_fallback will be True
# and result.model_tier will be SONNET

Metrics Recording

Every call records Prometheus metrics:

  • klabautermann_llm_calls_total{model, purpose}
  • klabautermann_llm_tokens_total{model, token_type}
  • klabautermann_llm_call_latency_seconds{model}

Track costs, identify hot paths, optimize where it matters.

Environment Variables

Configure models without code changes:

# Override default model IDs
HAIKU_MODEL=claude-3-5-haiku-20241022
SONNET_MODEL=claude-sonnet-4-20250514
OPUS_MODEL=claude-opus-4-5-20251101

# Disable multi-model (use Sonnet everywhere)
ENABLE_MULTI_MODEL=false

Agent Default Mapping

For backward compatibility, agents have default model assignments:

from klabautermann.core import get_model_for_agent

get_model_for_agent("ingestor")     # -> Haiku
get_model_for_agent("researcher")   # -> Haiku
get_model_for_agent("orchestrator") # -> Sonnet
get_model_for_agent("executor")     # -> Sonnet

Priority Chain

Model selection follows this priority:

  1. Agent override (if configured)
  2. Purpose override (if configured)
  3. Complexity (if provided)
  4. Purpose default (if provided)
  5. Fallback tier (Sonnet)
