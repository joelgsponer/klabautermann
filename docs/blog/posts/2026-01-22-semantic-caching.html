Remember the Answer

"What's John's email?" Graph traversal. LLM call. 800 milliseconds.

"What's John's email?" Same question. Same answer. Another 800 milliseconds?

Not anymore.

The Cache

Hash the query. Store the result. Next time, return from memory:

from klabautermann.memory import (
    cache_search_results,
    get_cached_search_results,
)

# First time: compute and cache
results = await expensive_search(query)
cache_search_results(query, results)

# Second time: instant return
cached = get_cached_search_results(query)
if cached:
    return cached  # Microseconds, not milliseconds

Query Hashing

Queries normalize before hashing. Case doesn't matter. Whitespace doesn't matter:

from klabautermann.memory import hash_query

# These all produce the same hash
hash_query("What's John's email?")
hash_query("WHAT'S JOHN'S EMAIL?")
hash_query("  what's john's email?  ")

Parameters matter though:

from klabautermann.memory import hash_query_with_params

# Different params = different cache entries
hash_query_with_params("Find people", {"limit": 10})
hash_query_with_params("Find people", {"limit": 20})

TTL Expiration

Results expire. Default: 5 minutes. The knowledge graph changes, and stale answers become wrong answers:

# Configure via environment
SEMANTIC_CACHE_TTL=300  # 5 minutes default

# Or per entry
cache.set(query, results, ttl=60)  # 1 minute for this entry

LRU Eviction

When the cache fills up, drop the least recently used entries:

# Default: 1000 entries max
SEMANTIC_CACHE_MAX_ENTRIES=1000

# Oldest unused entries evicted first
# Recently accessed entries stay longer

Statistics

Track your cache performance:

from klabautermann.memory import get_search_cache_stats

stats = get_search_cache_stats()

print(f"Hit rate: {stats.hit_rate:.1%}")
print(f"Hits: {stats.hits}")
print(f"Misses: {stats.misses}")
print(f"Evictions: {stats.evictions}")
print(f"Current size: {stats.current_size}/{stats.max_size}")

Full API

from klabautermann.memory import (
    SemanticCache,         # The cache class
    hash_query,            # Hash a query string
    hash_query_with_params,# Hash query + parameters

    # Global cache convenience functions
    get_semantic_cache,    # Get singleton instance
    cache_search_results,  # Store results
    get_cached_search_results,  # Retrieve results
    invalidate_search_cache,    # Remove specific entry
    clear_search_cache,    # Clear all entries
    get_search_cache_stats,# Get statistics
)
