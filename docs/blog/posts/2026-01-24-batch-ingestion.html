Loading the Cargo Hold

Single episode ingestion works fine for real-time conversations. But what about importing conversation history? Migrating data from another system? Batch processing a folder of documents?

One at a time would take forever. The Ingestor now loads cargo in bulk.

The API

from klabautermann.agents.ingestor import Ingestor, BatchEpisode

episodes = [
    BatchEpisode(content="First conversation snippet"),
    BatchEpisode(content="Second conversation snippet"),
    BatchEpisode(content="Third conversation snippet", captain_uuid="user-123"),
]

result = await ingestor.batch_ingest(episodes, max_concurrent=5)

print(f"Processed: {result.successful}/{result.total}")
print(f"Success rate: {result.success_rate}%")

Parallel Processing

Episodes process concurrently, controlled by a semaphore. Default limit is 5 concurrent operations - enough to saturate the LLM API without overwhelming it.

semaphore = asyncio.Semaphore(max_concurrent)

async def process_episode(index, episode):
    async with semaphore:
        return await self._ingest_single_episode(index, episode, trace_id)

Tune max_concurrent based on your API limits and use case. Higher for throughput, lower for stability.

Independent Failures

One bad episode doesn't sink the batch. Each processes independently:

result = await ingestor.batch_ingest(episodes)

for r in result.results:
    if not r.success:
        print(f"Episode {r.index} failed: {r.error}")

Two of three failed? You still get one in The Locker. The result tells you exactly what happened to each episode.

Result Tracking

Three levels of detail:

@dataclass
class BatchIngestionResult:
    total: int          # How many we tried
    successful: int     # How many worked
    failed: int         # How many didn't
    results: list[EpisodeResult]  # Per-episode details

@dataclass
class EpisodeResult:
    index: int          # Position in batch
    success: bool       # Did it work?
    episode_name: str   # Graphiti's episode ID
    error: str          # What went wrong (if anything)

The to_dict() method serializes everything for logging or API responses.

Content Cleaning

Each episode goes through the same cleaning as single-episode ingestion:

  • Remove role prefixes ("User:", "Assistant:")
  • Strip roleplay markers (*actions*)
  • Clean up system mentions

Empty content after cleaning? That episode fails with "Empty content after cleaning" - other episodes continue.

Use Cases

  • History Import: Migrate conversation logs from another system
  • Document Processing: Ingest a folder of notes or documents
  • Backfill: Add historical data to an existing graph
  • Testing: Load synthetic data for development
