The Islands Name Themselves

The Cartographer finds islands. Now it describes them too.

AGENTS_EXTENDED.md Section 4.3 specified that communities should have AI-generated summaries. Today we made them speak.

The Problem (#75)

Knowledge Islands are clusters of related nodes - your work contacts, hobby activities, family connections. But a cluster without a label is just... a cluster. You need to know what's inside.

The Community model had a summary field. It was always None.

The Solution

Two new methods join the Cartographer:

async def generate_summaries(
    self,
    force_regenerate: bool = False,
    trace_id: str | None = None,
) -> dict[str, Any]:
    """Generate AI summaries for communities that need them."""

async def _generate_summary(
    self,
    community_name: str,
    community_theme: str,
    members: list[CommunityMember],
    trace_id: str | None = None,
) -> str:
    """Generate a summary for a single community."""

generate_summaries() finds all communities without summaries (or stale ones older than 1 week) and generates new ones.

The Summary Format

Summaries follow a consistent pattern:

Knowledge Island containing {theme}. Includes {composition}. Total: {count} nodes.

Example output:

Knowledge Island containing work-related entities. Includes 3 Persons, 2 Organizations, 1 Project. Total: 8 connected nodes.

The Implementation

For now, summaries use rule-based generation:

  • Count member nodes by label (Person, Organization, etc.)
  • Map theme to description (professional = "work-related entities")
  • Format composition (top 3 label types)

This can be upgraded to LLM-based summaries later. The infrastructure is ready - member descriptions are collected, just not used yet.

Staleness Detection

Summaries regenerate when:

  • summary IS NULL - never generated
  • last_updated < timestamp() - 604800000 - older than 1 week

Or force a complete refresh:

await cartographer.generate_summaries(force_regenerate=True)

Process Message Handler

Trigger via agent message:

msg = AgentMessage(
    source_agent="orchestrator",
    target_agent="cartographer",
    intent="cartographer_request",
    payload={"operation": "generate_summaries", "force_regenerate": False},
)

Knowledge Islands now introduce themselves. When you query communities, you see not just their theme and member count, but a description of what they contain.

An island without a name is just a point on the map. An island with a summary is a destination.
