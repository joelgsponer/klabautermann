Teaching the Bard to Choose

The Bard of the Bilge has a problem. With stories to tell and sagas to continue, how does it decide what to say?

Previously, the selection was simple: continue a saga if one exists, otherwise pick a random tidbit. But the spec demanded more nuance.

The Spec (#107, #108)

LORE_SYSTEM.md Section 4.1 defined 12 canonical standalone tidbits - one-liners that stand alone. Section 4.2 specified a weighted selection algorithm:

  • 30%: Continue an active saga
  • 20%: Start a new saga
  • 50%: Deliver a standalone tidbit

The goal: natural storytelling flow with variety.

The Algorithm

Weighted random selection with graceful fallbacks:

async def _select_and_add_tidbit(
    self,
    clean_response: str,
    channel: str | None = None,
    trace_id: str | None = None,
) -> SaltResult:
    """Select and add a tidbit based on weighted probabilities (#108)."""
    roll = random.random()
    continue_threshold = self.bard_config.continue_saga_weight
    start_threshold = continue_threshold + self.bard_config.start_saga_weight

    # 30% chance to continue an active saga
    if roll < continue_threshold:
        active_saga = await self._get_active_saga(trace_id=trace_id)
        if active_saga:
            try:
                tidbit, chapter = await self._continue_saga(
                    active_saga, channel=channel, trace_id=trace_id
                )
                return SaltResult(
                    salted_response=f"{clean_response}\n\n*{tidbit}*",
                    tidbit_added=True,
                    tidbit=tidbit,
                    is_continuation=True,
                    saga_id=active_saga.saga_id,
                    chapter=chapter,
                )
            except SagaLifecycleError:
                pass  # Fall through to standalone

    # 20% chance to start a new saga
    elif roll < start_threshold:
        try:
            tidbit, saga_id, chapter = await self.start_new_saga(
                channel=channel, trace_id=trace_id
            )
            return SaltResult(...)
        except SagaLifecycleError:
            pass  # Fall through to standalone

    # 50% standalone tidbit (or fallback)
    tidbit = self._generate_standalone_tidbit()
    return SaltResult(
        salted_response=f"{clean_response}\n\n*{tidbit}*",
        tidbit_added=True,
        tidbit=tidbit,
        is_continuation=False,
    )

The key insight: when saga operations fail (lifecycle limits, timing rules), fall through to standalone. Never fail to add a tidbit when one is requested.

Configurable Weights

BardConfig now has three weights:

# Tidbit selection weights from LORE_SYSTEM.md Section 4.2 (#108)
continue_saga_weight: float = 0.3  # 30% continue active saga
start_saga_weight: float = 0.2     # 20% start new saga
standalone_weight: float = 0.5     # 50% standalone tidbit

Want more sagas? Increase continue_saga_weight. Want pure variety? Set weights to 0/0/1.

The Test Challenge

Tests that expected saga continuation now had a 30% chance of failing. The fix: force the weights in test fixtures.

config = BardConfig(
    tidbit_probability=1.0,
    saga_continuation_probability=1.0,
    min_chapter_interval_hours=0,
    # Force saga continuation path in weighted selection (#108)
    continue_saga_weight=1.0,
    start_saga_weight=0.0,
    standalone_weight=0.0,
)

Deterministic tests need deterministic weights.
