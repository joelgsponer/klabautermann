The Hull Cleaner's Ledger

The Hull Cleaner scrapes barnacles - weak relationships, orphan messages, duplicate entities. It logs what it removes. But where did those logs go?

Into the void. Session-only. Restart the agent, and the history vanishes.

The Problem (#87)

AGENTS_EXTENDED.md Section 5 specified: "Persist audit trail of all pruning operations." The Hull Cleaner already generated AuditEntry objects - timestamp, action type, entity ID, reason. But they lived only in memory. No graph storage. No queryable history.

Compliance requires records. Debugging requires history. "What did you delete last month?" should have an answer.

The Solution

Store audit entries as Neo4j nodes. Query them like any other entity.

New Node Type

# core/ontology.py
class NodeLabel(str, Enum):
    # ... existing types ...
    AUDIT_LOG = "AuditLog"  # New

The Module

A new memory/audit_log.py with five functions:

save_audit_entry()      # Single entry
save_audit_entries()    # Batch with UNWIND
query_audit_log()       # Filter by time, action, entity type
get_audit_stats()       # Counts by action type
delete_old_audit_entries() # Cleanup

Batch Efficiency

The Hull Cleaner might prune hundreds of weak relationships in one run. Individual inserts would be slow. UNWIND solves this:

UNWIND $entries as entry
CREATE (al:AuditLog {
    uuid: entry.uuid,
    timestamp: entry.timestamp,
    action: entry.action,
    entity_type: entry.entity_type,
    entity_id: entry.entity_id,
    reason: entry.reason,
    agent_name: entry.agent_name,
    metadata: entry.metadata
})
RETURN al.uuid as uuid

One query. All entries. Fast.

Integration

The Hull Cleaner gains a new parameter:

await cleaner.scrape_barnacles(
    dry_run=False,
    persist_audit=True  # Store entries in graph
)

When persist_audit=True, after the operation completes, all audit entries flow into the graph as AuditLog nodes.

Query Patterns

What did the Hull Cleaner remove last week?

entries = await cleaner.query_stored_audit(
    filters={
        "start_time": "2026-01-17T00:00:00",
        "end_time": "2026-01-24T00:00:00",
        "action_types": ["DELETE_RELATIONSHIP", "DELETE_NODE"]
    }
)
