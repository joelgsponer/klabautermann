The Cartographer's Weekly Patrol

The Cartographer finds Knowledge Islands. But islands shift. New entities join. Relationships form. Without regular surveys, the map grows stale.

AGENTS_EXTENDED.md Section 4 specified scheduled community detection. Today we made the Cartographer sail on a schedule.

The Problem (#76)

The Cartographer's detect_communities() runs Louvain clustering to identify Knowledge Islands. But it only runs when explicitly called. As you add contacts, projects, and notes throughout the week, the islands don't update themselves.

You could ask: "What's on my Professional Island?" and get outdated results.

The Solution

Register the Cartographer with APScheduler:

async def cartographer_job() -> None:
    job_trace_id = str(uuid.uuid4())
    logger.info("[BEACON] Scheduled Cartographer community detection triggered")
    try:
        await cartographer.detect_communities(trace_id=job_trace_id)
    except Exception as e:
        logger.error(f"[STORM] Community detection failed: {e}")
        # Don't re-raise - let scheduler continue

The Schedule

Default: Sunday midnight (cron: 0 0 * * 0)

Weekly is enough. Community structures change slowly. Running more frequently wastes resources. Running less frequently lets the map drift too far from reality.

Configuration

Like other scheduled jobs, the Cartographer is configurable:

config = {
    "cartographer": {
        "enabled": True,        # Toggle the job
        "cron": "0 2 * * 1"    # Custom schedule (Monday 2am)
    }
}

Graceful Failure

The Louvain algorithm can fail. GDS might not be available. The graph might be empty. When it fails:

  • Log the error at STORM level
  • Don't re-raise - scheduler continues
  • Next week's run tries again

No crashes. No manual intervention. The ship sails on.
