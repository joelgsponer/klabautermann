Scraping Barnacles: Duplicate Detection

Over time, knowledge graphs accumulate duplicates. "John Doe" from one conversation becomes "Jon Doe" in another. "Acme Corp" morphs into "Acme Corporation" from a different email thread.

The Hull Cleaner now spots these doubles and merges them.

The Fuzzy Matching Problem

Exact string matching won't catch duplicates. Names vary across contexts:

  • "John Doe" vs "Jon Doe" (typo)
  • "Sarah Smith" vs "Sarah J. Smith" (middle initial)
  • "Acme Corp" vs "Acme Corporation" (abbreviation)

We need similarity matching that tolerates small variations while rejecting completely different entities.

Levenshtein Similarity

The Hull Cleaner uses APOC's Levenshtein similarity function to compare entity names:

MATCH (p1:Person), (p2:Person)
WHERE p1.uuid < p2.uuid
  AND apoc.text.levenshteinSimilarity(
      toLower(p1.name),
      toLower(p2.name)
  ) > 0.85
RETURN p1.uuid, p2.uuid, p1.name, p2.name

Levenshtein distance counts the minimum edits needed to transform one string into another. The similarity score (0.0 to 1.0) tells us how close two names are.

Confidence Levels

Not all matches deserve the same treatment:

from klabautermann.agents import DuplicateCandidate

candidate = DuplicateCandidate(
    uuid1="uuid-1",
    uuid2="uuid-2",
    name1="John Doe",
    name2="John D.",
    entity_type="Person",
    similarity=0.86,
    email_match=True,
)

print(candidate.confidence)  # "HIGH"
  • HIGH: Similarity >= 0.95 OR matching email. Auto-merge.
  • MEDIUM: Similarity 0.85-0.95. Flag for review.
  • LOW: Below 0.85. Skip entirely.

APOC Refactor Merge

Once we're confident two nodes are duplicates, APOC handles the merge:

MATCH (keep {uuid: $keep_uuid}), (remove {uuid: $remove_uuid})
CALL apoc.refactor.mergeNodes([keep, remove], {
    properties: 'combine',
    mergeRels: true
})
YIELD node
RETURN node.uuid

The merge:

  • Combines properties: Both nodes' attributes are kept
  • Transfers relationships: All edges from the removed node move to the kept node
  • Preserves UUIDs: The kept node's UUID remains stable

Integration with Scrape Barnacles

Duplicate detection is now part of the Hull Cleaner's regular maintenance:

from klabautermann.agents import HullCleaner, HullCleanerConfig

config = HullCleanerConfig(
    detect_duplicates=True,
    duplicate_similarity_threshold=0.85,
    duplicate_auto_merge_threshold=0.95,
)

cleaner = HullCleaner(neo4j_client=neo4j, config=config)

# Full maintenance including duplicate detection
result = await cleaner.scrape_barnacles(dry_run=False)

print(f"Merged: {result.nodes_removed}")

The Review Flow

MEDIUM confidence matches get flagged in the audit trail:

# Find duplicates first
duplicates = await cleaner.find_duplicate_entities()

# Review MEDIUM confidence ones
for dup in duplicates:
    if dup.confidence == "MEDIUM":
        print(f"Review: '{dup.name1}' vs '{dup.name2}'")
        print(f"  Similarity: {dup.similarity:.0%}")
        print(f"  Entity type: {dup.entity_type}")

# Then merge with review complete
result = await cleaner.merge_duplicates(
    dry_run=False,
    auto_merge_only=False  # Include MEDIUM
)
