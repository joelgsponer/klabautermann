Wiring Up the Bard

The Bard of the Bilge has been telling tales for a while now. Saga lifecycle management keeps the stories from sprawling forever. But there was one problem: the Bard existed in isolation. It could salt responses, but nothing ever asked it to.

Time to plug it in.

Three Issues, One Integration

The work split naturally into three pieces:

  • #116: Create the bard.yaml config file
  • #117: Add Pydantic models for Bard configuration
  • #109: Wire salt_response into the Orchestrator

Config first, types second, integration last. The classic bottom-up approach.

The Configuration File (#116)

Agents in Klabautermann are configured via YAML files in config/agents/. The Bard needed its own:

# config/agents/bard.yaml
name: bard

model:
  primary: claude-3-5-haiku-20241022
  temperature: 0.8
  max_output_tokens: 200

# 7% chance to add a tidbit
tidbit_probability: 0.07

# 30% chance to continue an existing saga
saga_continuation_probability: 0.3

saga_rules:
  max_chapters: 5
  max_active: 3
  timeout_days: 30
  min_interval_hours: 1.0

storm_mode:
  enabled: true
  keywords:
    - urgent
    - emergency
    - critical
    - asap

display:
  format: italic
  separator: "\n\n"

The Bard uses Haiku for generation - fast, cheap, and whimsical enough for tidbits. Temperature 0.8 keeps things creative. Storm mode keywords disable tidbits when things get serious.

The Pydantic Models (#117)

YAML is nice for humans. Pydantic is nice for Python. The config manager needed types:

class SagaRulesConfig(BaseModel):
    """Saga lifecycle rules configuration."""

    model_config = ConfigDict(extra="forbid")

    max_chapters: int = Field(default=5, ge=1, le=20)
    max_active: int = Field(default=3, ge=1, le=10)
    timeout_days: int = Field(default=30, ge=1, le=365)
    min_interval_hours: float = Field(default=1.0, ge=0.0, le=24.0)


class StormModeConfig(BaseModel):
    """Storm mode configuration."""

    model_config = ConfigDict(extra="forbid")

    enabled: bool = True
    keywords: list[str] = Field(
        default_factory=lambda: ["urgent", "emergency", "critical", "asap"]
    )


class BardConfig(BaseModel):
    """Bard of the Bilge configuration."""

    model_config = ConfigDict(extra="allow")

    name: str = "bard"
    model: ModelConfig = Field(default_factory=ModelConfig)
    tidbit_probability: float = Field(default=0.07, ge=0.0, le=1.0)
    saga_continuation_probability: float = Field(default=0.3, ge=0.0, le=1.0)
    saga_rules: SagaRulesConfig = Field(default_factory=SagaRulesConfig)
    storm_mode: StormModeConfig = Field(default_factory=StormModeConfig)
    display: LoreDisplayConfig = Field(default_factory=LoreDisplayConfig)

Field constraints catch invalid configs at load time: probabilities between 0 and 1, chapter limits between 1 and 20, timeouts between 1 and 365 days. The config manager now knows how to parse bard.yaml.

The Integration (#109)

The Orchestrator needed three things: initialization, storm mode detection, and the actual salting.

Initialization

A new captain_uuid parameter tells the Orchestrator who's at the helm:

def __init__(
    self,
    graphiti: GraphitiClient | None = None,
    thread_manager: ThreadManager | None = None,
    neo4j_client: Any | None = None,
    config: dict[str, Any] | None = None,
    captain_uuid: str | None = None,  # NEW
) -> None:
    ...
    self.captain_uuid = captain_uuid

    # Initialize Bard for response salting
    self._bard: BardOfTheBilge | None = None
    self._bard_config = self._load_bard_config()

    if neo4j_client and captain_uuid and self._bard_config.get("enabled", True):
        self._bard = BardOfTheBilge(
            neo4j_client=neo4j_client,
            captain_uuid=captain_uuid,
            config=BardAgentConfig(**self._bard_config),
        )
        self._agent_registry["bard"] = self._bard

No captain UUID? No Bard. No Neo4j client? No Bard. Bard disabled in config? No Bard. The Orchestrator works fine without the storyteller.

Storm Mode Detection

When things are urgent, the Bard stays quiet:

def _detect_storm_mode(self, response: str) -> bool:
    """Check if response contains storm mode keywords."""
    storm_config = self._bard_config.get("storm_mode", {})
    if not storm_config.get("enabled", True):
        return False

    keywords = storm_config.get("keywords", ["urgent", "emergency", "critical", "asap"])
    response_lower = response.lower()
    return any(keyword.lower() in response_lower for keyword in keywords)

Case-insensitive matching. "URGENT" and "urgent" both trigger storm mode.

The Salting

The _apply_personality method now calls the Bard:

async def _apply_personality(self, response: str, trace_id: str) -> str:
    """Apply personality transformations to response."""
    if self._bard is None:
        return response

    storm_mode = self._detect_storm_mode(response)

    try:
        salt_result = await self._bard.salt_response(
            clean_response=response,
            storm_mode=storm_mode,
            trace_id=trace_id,
        )

        if salt_result.tidbit_added:
            logger.debug(
                f"[CHART] Bard added tidbit",
                extra={
                    "saga_id": salt_result.saga_id,
                    "chapter": salt_result.chapter,
                    "is_continuation": salt_result.is_continuation,
                },
            )
            return salt_result.salted_response

        return response

    except Exception as e:
        logger.warning(f"[SWELL] Bard error, returning original: {e}")
        return response

The key insight: graceful degradation. If the Bard errors - database issues, LLM timeouts, whatever - we log a warning and return the original response. The user never sees an error. They just don't get a tidbit.

The Result

Pass a captain_uuid to the Orchestrator, and about 7% of responses will now include a tale from the digital seas. The Bard knows when to stay quiet (storm mode), knows when to continue a saga (30% chance), and fails gracefully when something goes wrong.

Your assistant has personality now. Configurable. Optional. Graceful.
