Following the Thread

A message arrives. The Orchestrator routes it. The Researcher queries the graph. The Executor calls an API. Somewhere along the way, something fails. Where?

Without tracing, you grep logs hoping timestamps line up. With tracing, you follow one ID through the entire request path.

W3C Trace Context

The standard for distributed tracing. A 128-bit trace ID identifies the entire request journey. 64-bit span IDs mark individual operations. Parent relationships create a tree.

from klabautermann.core import trace_span, current_trace_id

# Start a trace
with trace_span("orchestrator.process") as span:
    span.set_attribute("intent", "search")
    span.set_attribute("channel", "telegram")

    # Nested spans share the trace ID
    with trace_span("researcher.query") as child:
        child.set_attribute("query", "John Acme")
        results = await researcher.search(...)
        child.set_attribute("result_count", len(results))

    # Check the current trace ID anywhere
    print(f"Trace: {current_trace_id()}")

Spans

A span represents a unit of work. It has a name, timing, status, and attributes. Spans nest to show operation hierarchy.

span = Span(
    name="llm.call",
    trace_id="4bf92f3577b34da6a3ce929d0e0e4736",
    span_id="00f067aa0ba902b7",
    parent_span_id="a3ce929d0e0e4736",
)

span.set_attribute("model", "claude-3-5-sonnet")
span.set_attribute("tokens", 1523)
span.add_event("response.received", {"latency_ms": 847})

# When done
span.end(SpanStatus.OK)

Error Tracking

When something fails, the span records it:

try:
    await external_api.call()
except Exception as e:
    span.set_error(e)
    raise

# Span now contains:
# status: ERROR
# error.type: "ConnectionError"
# error.message: "Connection refused"

Context Propagation

Traces propagate via W3C traceparent headers when crossing service boundaries:

# Parse incoming header
ctx = TraceContext.from_traceparent(
    "00-4bf92f3577b34da6a3ce929d0e0e4736-00f067aa0ba902b7-01"
)

# Generate outgoing header
outgoing = ctx.to_traceparent(span.span_id)
# "00-4bf92f3577b34da6a3ce929d0e0e4736-{new_span_id}-01"

The trace ID stays constant. Span IDs change. Parent relationships form.

Export Options

Two export modes for different environments:

# Development: log to stdout
TRACING_ENABLED=true
TRACING_EXPORT=stdout

# Production: send to OpenTelemetry collector
TRACING_ENABLED=true
TRACING_EXPORT=otlp
OTEL_EXPORTER_OTLP_ENDPOINT=http://collector:4317

The Tracer

A global tracer instance manages context:

from klabautermann.core import get_tracer, start_trace

# Get the singleton tracer
tracer = get_tracer()

# Start a new trace manually
ctx = start_trace("request")
print(f"New trace: {ctx.trace_id}")

# Check current state
current_span = tracer.get_current_span()
current_ctx = tracer.get_current_context()

Configuration

Environment variables control tracing behavior:

# Enable/disable tracing (default: true)
TRACING_ENABLED=true

# Export destination: none, stdout, otlp
TRACING_EXPORT=stdout

# Service name for traces
OTEL_SERVICE_NAME=klabautermann

# Verbose span output to stdout
TRACE_VERBOSE=true
