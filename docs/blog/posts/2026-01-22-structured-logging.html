Logs That Tell Stories

Text logs are for humans. JSON logs are for machines. Today we enhanced the logging system to speak both languages fluently.

The Problem

When you're debugging locally, you want readable output:

15:42:03 | abc12345 | researcher   | [CHART]     | Processing query

When you're running in production with Elasticsearch or Loki, you need structure:

{
  "timestamp": "2026-01-22T15:42:03.123Z",
  "level": "INFO",
  "service": "klabautermann",
  "trace_id": "abc12345",
  "message": "Processing query"
}

Same information, different format. The logging system now supports both.

Enable JSON Output

export LOG_FORMAT=json

That's it. Every log becomes a JSON line ready for aggregation.

Standard Fields

Every JSON log entry includes:

{
  "timestamp": "2026-01-22T15:42:03.123456+00:00",
  "level": "INFO",
  "nautical_level": "[CHART]",
  "logger": "klabautermann.agents.researcher",
  "message": "Planning search strategy",
  "service": "klabautermann",
  "environment": "dev",
  "hostname": "prod-node-1"
}

Service Identity

export LOG_SERVICE_NAME=klabautermann-worker-1
export LOG_ENVIRONMENT=prod

When running multiple instances, these fields help you filter and aggregate.

Context Without Repetition

Instead of passing trace_id to every log call:

# Tedious
logger.info("Start", extra={"trace_id": tid})
logger.info("Middle", extra={"trace_id": tid})
logger.info("End", extra={"trace_id": tid})

Use the context manager:

from klabautermann.core.logger import log_context

with log_context(trace_id=tid, user_id=uid):
    logger.info("Start")   # Includes trace_id and user_id
    logger.info("Middle")  # Same context
    logger.info("End")     # Same context

The context is stored in a contextvars.ContextVar, so it's async-safe and thread-safe.

Structured Exceptions

Exceptions become structured data:

{
  "timestamp": "2026-01-22T15:42:03.123Z",
  "level": "ERROR",
  "message": "Failed to process",
  "exception": {
    "type": "ValueError",
    "message": "Invalid input",
    "traceback": "Traceback (most recent call last):..."
  }
}

Type and message are separate fields - easy to aggregate error types and search for specific messages.

Source Location

For deep debugging, enable source location:

export LOG_INCLUDE_SOURCE=true

This adds:

{
  "source": {
    "file": "/app/src/klabautermann/agents/researcher.py",
    "line": 245,
    "function": "_plan_search"
  }
}

Expensive in terms of log size, but invaluable when hunting a bug.

The Implementation

The enhanced JSONFormatter:

class JSONFormatter(logging.Formatter):
    """JSON formatter for structured log aggregation."""

    # Cache static fields (computed once)
    _static_fields: ClassVar[dict[str, str] | None] = None

    @classmethod
    def _get_static_fields(cls) -> dict[str, str]:
        if cls._static_fields is None:
            cls._static_fields = {
                "service": os.getenv("LOG_SERVICE_NAME", "klabautermann"),
                "environment": os.getenv("LOG_ENVIRONMENT", "dev"),
                "hostname": platform.node(),
            }
        return cls._static_fields

    def format(self, record: LogRecord) -> str:
        log_entry = self._get_static_fields().copy()

        # Core fields
        log_entry.update({
            "timestamp": datetime.fromtimestamp(record.created, tz=UTC).isoformat(),
            "level": record.levelname,
            "nautical_level": NauticalFormatter.LEVEL_MAP.get(record.levelname, record.levelname),
            "logger": record.name,
            "message": record.getMessage(),
        })

        # Context from log_context()
        context = get_log_context()
        if context:
            log_entry.update(context)

        # Fields from record.extra
        for key in ("trace_id", "agent_name", "latency_ms", ...):
            value = getattr(record, key, None)
            if value is not None:
                log_entry[key] = value

        return json.dumps(log_entry, default=str)

Static fields are computed once and cached. Context fields are merged on every call. Record fields are extracted dynamically.

Context Manager

_log_context: contextvars.ContextVar[dict[str, Any]] = contextvars.ContextVar(
    "log_context", default={}
)

@contextmanager
def log_context(**kwargs: Any) -> Iterator[None]:
    token = set_log_context(**kwargs)
    try:
        yield
    finally:
        _log_context.reset(token)

The token pattern ensures nested contexts work correctly - each scope restores the previous state on exit.
