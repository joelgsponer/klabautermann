The Guest List

TheSieve filters your inbox. It catches noise, blocks spam, and detects prompt injection attempts. But sometimes the filter catches things you actually want.

Your CEO sends a one-word email: "Approved." TheSieve sees a short message with no context and flags it as noise.

Now there's a guest list.

The VIP Whitelist

VIPs bypass TheSieve's noise and content filters. They still go through security checks - even the CEO doesn't get to inject prompts - but their short emails, newsletters, and automated messages all pass through.

sieve = TheSieve(
    min_words=5,
    vip_whitelist=["ceo@company.com", "@company.com"]
)

Matching Logic

Two kinds of matches:

  • Exact match: alice@example.com matches only that address
  • Domain wildcard: @company.com matches anyone at that domain

Both are case-insensitive. Both handle "Name <email>" format automatically.

def _is_vip(self, sender: str) -> bool:
    if not self._vip_whitelist or not sender:
        return False

    sender_lower = sender.lower()

    # Handle "Name " format
    if "<" in sender_lower and ">" in sender_lower:
        start = sender_lower.find("<") + 1
        end = sender_lower.find(">")
        sender_lower = sender_lower[start:end]

    for vip in self._vip_whitelist:
        if vip.startswith("@") and sender_lower.endswith(vip):
            return True
        if sender_lower == vip:
            return True

    return False

Security First

The order matters. In filter_email():

  1. Security checks (prompt injection detection)
  2. VIP whitelist check
  3. Noise filters (min words, promotional content)

VIPs skip step 3, never step 1. A compromised VIP account attempting prompt injection still gets blocked.

# 1. Security checks FIRST - VIPs don't bypass these
if is_prompt_injection:
    return EmailManifest(
        is_manifest_worthy=False,
        risk_level=RiskLevel.CRITICAL,
        filter_reason="Prompt injection detected",
    )

# 2. VIP whitelist check - skip noise/content filters but NOT security
if self._is_vip(sender):
    return EmailManifest(
        is_manifest_worthy=True,
        risk_level=RiskLevel.LOW,
        filter_reason="VIP whitelist bypass",
    )

Runtime Management

Add and remove VIPs without restarting:

sieve.add_vip("partner@external.com")
sieve.remove_vip("former-ceo@company.com")
print(sieve.vip_whitelist)  # Current list

Configuration

Set VIPs in PurserConfig:

@dataclass
class PurserConfig:
    gmail_credentials_path: str | None = None
    calendar_credentials_path: str | None = None
    vip_whitelist: list[str] = field(default_factory=list)

Pass them when creating the Purser, and they flow through to TheSieve.
