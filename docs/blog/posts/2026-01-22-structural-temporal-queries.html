Charting the Hierarchy and Sailing Through Time

Knowledge graphs store relationships, but navigating them takes more than simple lookups. Two common patterns emerged in our use cases:

  1. Hierarchical traversal: "Who does Sarah report to?" or "Who reports to John?"
  2. Historical queries: "Who did Sarah work for last year?"

The Researcher agent now handles both.

Traversing Reporting Chains

Organizations have hierarchies. REPORTS_TO relationships form chains from individual contributors up through managers to executives. Traversing these chains is a common query pattern:

from klabautermann.memory import (
    traverse_reporting_chain,
    TraversalDirection,
)

# Find Sarah's managers (upward chain)
result = await traverse_reporting_chain(
    client=neo4j_client,
    person_name="Sarah",
    direction=TraversalDirection.OUTGOING,  # Upward
    max_depth=5,
)

for node in result.nodes:
    print(f"Level {node['depth']}: {node['name']}")

Output might show:

Level 1: John Manager
Level 2: Sarah VP
Level 3: Chris CEO

Finding Direct Reports

Flip the direction to find who reports to someone:

# Find John's direct reports (downward chain)
result = await traverse_reporting_chain(
    client=neo4j_client,
    person_uuid="john-uuid",
    direction=TraversalDirection.INCOMING,  # Downward
    max_depth=3,
)

for node in result.nodes:
    print(f"  {'  ' * (node['depth'] - 1)}{node['name']}")

The Temporal Challenge

Relationships change over time. Sarah worked at Acme Corp in 2024, but joined StartupXYZ in 2025. A simple "Where does Sarah work?" doesn't capture history.

Graphiti's temporal model stores created_at and expired_at timestamps on relationships. Our new temporal module parses natural language time expressions and filters accordingly.

Parsing Time Expressions

The parse_time_expression() function extracts temporal context from queries:

from klabautermann.memory import parse_time_expression

# Relative past
result = parse_time_expression("meetings last week")
print(result.expression_type)  # RELATIVE_PAST
print(result.start)            # 2026-01-15 00:00:00
print(result.end)              # 2026-01-22 00:00:00

# Absolute year
result = parse_time_expression("who worked at Acme in 2024")
print(result.expression_type)  # ABSOLUTE
print(result.start)            # 2024-01-01 00:00:00
print(result.end)              # 2024-12-31 23:59:59

# Point-in-time snapshot
result = parse_time_expression("as of last year")
print(result.expression_type)  # AS_OF
print(result.as_of)            # 2025-12-31 23:59:59

Supported Expressions

The parser handles a variety of time expressions:

  • Relative past: "yesterday", "last week", "last month", "3 days ago", "2 weeks ago"
  • Relative future: "tomorrow", "next week", "in 5 days"
  • Absolute: "in 2024", "2025"
  • As-of snapshots: "as of last year", "as of last month"
  • Informal: "recently", "soon"

Historical Relationship Queries

The real power comes from querying relationships at a point in time:

from datetime import datetime
from zoneinfo import ZoneInfo
from klabautermann.memory import get_historical_relationships

# Who did Sarah work for last year?
as_of = datetime(2025, 12, 31, tzinfo=ZoneInfo("UTC"))

results = await get_historical_relationships(
    client=neo4j_client,
    person_name="Sarah",
    relationship_type="WORKS_AT",
    as_of=as_of,
)

for rel in results:
    print(f"Sarah worked at {rel['target_name']}")
    print(f"  From: {rel['since']} to {rel['until']}")

The query filters by:

  • created_at <= as_of: Relationship existed by that time
  • expired_at IS NULL OR expired_at > as_of: Relationship hadn't ended yet

Combining with Temporal Filtering

The reporting chain traversal respects temporal filters too:

# Who did Sarah report to in 2024?
result = await traverse_reporting_chain(
    client=neo4j_client,
    person_name="Sarah",
    direction=TraversalDirection.OUTGOING,
    include_expired=True,  # Include historical relationships
)

# Filter results manually for 2024
as_of = datetime(2024, 12, 31, tzinfo=ZoneInfo("UTC"))

The Cypher Behind It

Historical queries use Neo4j's timestamp comparisons:

MATCH (p:Person)-[r]->(target)
WHERE toLower(p.name) CONTAINS toLower($person_name)
  AND type(r) = $rel_type
  AND r.created_at <= $as_of_ms
  AND (r.expired_at IS NULL OR r.expired_at > $as_of_ms)
RETURN p.name, type(r), target.name, r.created_at, r.expired_at
ORDER BY r.created_at DESC

Timestamps are stored as milliseconds since epoch for efficient indexing.
