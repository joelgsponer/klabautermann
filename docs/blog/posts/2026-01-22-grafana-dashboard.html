Eyes on the Dashboard

Yesterday we added Prometheus metrics. Today we give them a face.

One Command

docker-compose -f docker-compose.monitoring.yml up -d

That's it. Prometheus starts scraping. Grafana comes up with dashboards pre-loaded. Visit http://localhost:3000 and you're watching your ship.

What You See

Four sections. Each tells a different story.

Agent Performance

The crew at work:

  • Request Rate: Messages per second flowing to each agent
  • Latency (p50/p95): How long agents take to respond
  • Error Rate: Gauge that turns yellow at 1%, red at 5%
  • Status: Green for running, red for stopped
  • Inbox Size: Messages waiting in each queue

API Performance

The bridge communications:

  • Request Rate: HTTP requests by endpoint and method
  • Latency (p50/p95): Response times for each route
  • Status Codes: Stacked bars - green (2xx), orange (4xx), red (5xx)
  • WebSocket Connections: Active TUI clients

LLM Performance

The oracle's workload:

  • Call Rate: LLM calls by model and purpose
  • Latency: Response times by model (Haiku vs Sonnet vs Opus)
  • Token Usage: Input and output token rates

Graph Operations

The memory systems:

  • Operation Rate: Searches, episode additions, entity lookups
  • Latency: Graph query response times

Template Variables

Filter what you see:

  • $agent: Select specific agents or all
  • $endpoint: Focus on specific API routes
  • $datasource: Switch between Prometheus instances

Zero Configuration

Grafana provisions everything on startup:

# Dashboard provisioning
providers:
  - name: 'Klabautermann'
    folder: 'Klabautermann'
    type: file
    options:
      path: /var/lib/grafana/dashboards

# Datasource provisioning
datasources:
  - name: Prometheus
    type: prometheus
    url: http://prometheus:9090
    isDefault: true

No clicking through UIs. No manual imports. Start the containers, open the browser, see the data.

Query Examples

The PromQL behind the panels:

# Agent request rate
rate(klabautermann_agent_requests_total{agent_name=~"$agent"}[$__rate_interval])

# Agent latency p95
histogram_quantile(0.95,
  sum(rate(klabautermann_agent_request_latency_ms_bucket{agent_name=~"$agent"}[$__rate_interval]))
  by (le, agent_name)
)

# Agent error rate
sum(rate(klabautermann_agent_errors_total[$__rate_interval])) by (agent_name)
/
sum(rate(klabautermann_agent_requests_total[$__rate_interval])) by (agent_name)

# API requests by status
sum(rate(klabautermann_api_requests_total[$__rate_interval])) by (status_code)
