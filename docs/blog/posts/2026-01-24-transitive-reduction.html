Pruning the Triangles

Graphs accumulate shortcuts. You meet Alice. Alice introduces you to Bob. Bob introduces you to Charlie. Then you meet Charlie directly at a conference. Now your graph has:

  • You -> Alice -> Bob -> Charlie (the original chain)
  • You -> Charlie (the direct connection)

The intermediate path through Bob? Redundant. The Hull Cleaner now finds and prunes it.

The Problem (#86)

AGENTS_EXTENDED.md specified transitive path reduction as a pruning rule. The concept: when A->C exists directly AND A->B->C exists indirectly, evaluate whether the indirect path adds value. If the direct connection is stronger (higher weight), the intermediate hop is noise.

Without pruning, the graph bloats with redundant paths that slow queries and clutter results.

The Algorithm

Find triangles where the direct path is at least as strong as the indirect:

MATCH (a)-[r1]->(b)-[r2]->(c)
WHERE (a)-[direct]->(c)
AND type(r1) = type(r2) = type(direct)
AND direct.weight >= (r1.weight + r2.weight) / 2
RETURN a, b, c, id(r2) as redundant_rel_id

We compare the direct weight against the average of the two-hop path. If direct is stronger, the intermediate relationship (r2) is marked for removal.

The Implementation

@dataclass
class TransitivePath:
    source_uuid: str
    intermediate_uuid: str
    target_uuid: str
    source_name: str
    intermediate_name: str
    target_name: str
    relationship_type: str
    direct_weight: float
    indirect_weight: float
    redundant_rel_id: int

Configuration

config = HullCleanerConfig(
    detect_transitive_redundancy=True,   # Enable/disable
    transitive_weight_threshold=0.5,     # Comparison threshold
    max_transitive_per_run=100,          # Limit per operation
)

Usage

# Find redundant paths
paths = await cleaner.find_transitive_paths()

# Preview what would be pruned
result = await cleaner.reduce_transitive_paths(dry_run=True)

# Actually prune them
result = await cleaner.reduce_transitive_paths(dry_run=False)

Integration

Transitive reduction is now step 4 in scrape_barnacles():

  1. Prune weak relationships
  2. Remove orphan messages
  3. Merge duplicate entities
  4. Reduce transitive paths (new)
  5. Persist audit log (if enabled)

Audit Trail

Two new PruningAction values:

  • TRANSITIVE_PREVIEW - Dry run mode
  • TRANSITIVE_REDUCTION - Actual deletion

Each audit entry captures the triangle: source, intermediate, and target nodes, plus the weight comparison that justified the pruning.
