Querying the Bard's Memory

The Bard of the Bilge has been telling tales. Sagas span multiple chapters. Stories weave across CLI and Telegram. But until now, there was no way to look back at what was told.

Time to give the Bard a memory you can query.

Four Queries, One Story

The LORE_SYSTEM spec outlined four query patterns in Section 6. We implemented all four:

  • #112: Get recent lore - what tales were told lately?
  • #113: Get saga chain - every chapter of a saga, in order
  • #114: Cross-channel story - where did the saga travel?
  • #115: Saga statistics - how many tales per Captain?

Recent Lore (#112)

Simple but useful: "What stories have I heard lately?"

async def get_recent_lore(
    self,
    limit: int = 5,
    trace_id: str | None = None,
) -> list[LoreEpisode]:
    """Get the most recent story episodes told to this Captain."""
    query = """
    MATCH (le:LoreEpisode)-[:TOLD_TO]->(p:Person {uuid: $captain_uuid})
    RETURN le.uuid as uuid, le.saga_id as saga_id, le.saga_name as saga_name,
           le.chapter as chapter, le.content as content, le.channel as channel,
           le.told_at as told_at, le.created_at as created_at,
           p.uuid as captain_uuid
    ORDER BY le.told_at DESC
    LIMIT $limit
    """

Ordered by told_at DESC. Most recent first.

Saga Chain (#113)

Get all chapters of a saga in order. We already had _get_saga_chapters internally - this just exposes it publicly:

async def get_saga_chain(
    self,
    saga_id: str,
    trace_id: str | None = None,
) -> list[LoreEpisode]:
    """Get all chapters of a saga in order."""
    return await self._get_saga_chapters(
        saga_id,
        limit=self.bard_config.max_saga_chapters,
        trace_id=trace_id,
    )

Ordered by chapter ASC. Chapter 1 first, then 2, then 3.

Cross-Channel Story (#114)

This is the interesting one. Show how a saga traveled across channels:

async def get_cross_channel_story(
    self,
    saga_id: str,
    trace_id: str | None = None,
) -> list[dict[str, Any]]:
    """Show how a story traveled across channels."""
    query = """
    MATCH (le:LoreEpisode {saga_id: $saga_id})-[:TOLD_TO]->(p:Person {uuid: $captain_uuid})
    WITH le ORDER BY le.chapter ASC
    RETURN le.chapter as chapter, le.content as content,
           le.channel as channel, le.told_at as told_at,
           le.saga_name as saga_name
    """

The result shows the journey:

[
  {"chapter": 1, "channel": "cli", ...},      # Started on CLI
  {"chapter": 2, "channel": "telegram", ...}, # Continued on Telegram
  {"chapter": 3, "channel": "cli", ...}       # Back to CLI
]

Stories that span channels - that's the Captain-context pattern in action.

Saga Statistics (#115)

Admin-level view: how many tales has each Captain heard?

async def get_saga_statistics_by_captain(
    self,
    trace_id: str | None = None,
) -> list[dict[str, Any]]:
    """Count sagas and chapters grouped by Captain."""
    query = """
    MATCH (le:LoreEpisode)-[:TOLD_TO]->(p:Person)
    WITH p.uuid as captain_uuid, p.name as captain_name,
         le.saga_id as saga_id, count(le) as chapter_count
    WITH captain_uuid, captain_name,
         count(DISTINCT saga_id) as total_sagas,
         sum(chapter_count) as total_chapters
    RETURN captain_uuid, captain_name, total_sagas, total_chapters
    ORDER BY total_chapters DESC
    """

Unlike the other queries, this one isn't scoped to a single Captain. It aggregates across all Captains - useful for analytics.
