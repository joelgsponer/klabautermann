Taking the Pulse

The Purser syncs your Gmail and Calendar. But how do you know if those syncs are actually working?

Until now, you didn't. Now you do.

The Problem

External services fail. Network hiccups. API rate limits. OAuth token expiration. The sync might have been running "successfully" while silently dropping data.

Operations need observability. The Purser needed a health check.

SyncHealthMetrics

Each service now tracks its sync history:

from klabautermann.agents import Purser, SyncService

purser = Purser(neo4j_client=neo4j)

# Get health for all services
health = purser.get_sync_health()

print(health["overall_healthy"])  # True or False
print(health["health"]["gmail"]["success_rate"])  # 100.0
print(health["health"]["calendar"]["sync_count"])  # 42

Every metric you need in one call.

What Gets Tracked

The SyncHealthMetrics dataclass captures:

  • sync_count: Total sync operations attempted
  • success_count: Syncs that completed without errors
  • failure_count: Syncs that failed
  • success_rate: Percentage (calculated property)
  • is_healthy: True if success_rate >= 90%
  • last_success: Timestamp of last successful sync
  • last_failure: Timestamp of last failure
  • last_error: Most recent error message
  • total_items_synced: Cumulative items processed
  • avg_duration_ms: Rolling average sync duration

Health Threshold

A service is considered healthy at 90% or higher success rate:

@property
def is_healthy(self) -> bool:
    """Determine if sync is healthy (>= 90% success rate)."""
    return self.success_rate >= 90.0

Nine out of ten syncs need to succeed. That's the bar.

Automatic Recording

Every sync operation now records its outcome:

# After sync_gmail() completes:
if errors:
    self._health_metrics[SyncService.GMAIL].record_failure(
        errors[0], duration_ms
    )
else:
    self._health_metrics[SyncService.GMAIL].record_success(
        items_synced, duration_ms
    )

No manual tracking. Just run syncs; the metrics accumulate.

API Endpoint

Query health via the agent message interface:

message = AgentMessage(
    source_agent="orchestrator",
    target_agent="purser",
    intent="health",
    payload={"operation": "get_health"}
)

response = await purser.process_message(message)

# All services
print(response.payload["overall_healthy"])

# Specific service
message = AgentMessage(
    source_agent="orchestrator",
    target_agent="purser",
    intent="health",
    payload={"operation": "get_health", "service": "gmail"}
)

response = await purser.process_message(message)
print(response.payload["success_rate"])

Rolling Averages

Duration tracking uses a rolling window:

def _update_avg_duration(self, duration_ms: float) -> None:
    """Update rolling average duration."""
    if self._durations is None:
        self._durations = []
    self._durations.append(duration_ms)
    # Keep last 100 durations
    if len(self._durations) > 100:
        self._durations = self._durations[-100:]
    self.avg_duration_ms = sum(self._durations) / len(self._durations)

Recent performance matters more than ancient history.
