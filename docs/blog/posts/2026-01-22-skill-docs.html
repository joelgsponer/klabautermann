Self-Documenting Skills

Skills have metadata. That metadata is documentation waiting to happen. Today we added a generator that turns YAML frontmatter into proper documentation with zero manual effort.

The Problem

A skill definition contains everything you need to understand it:

---
name: summarize-thread
description: |
  Summarize any conversation or email thread.
  Use when user asks "summarize this thread", "TLDR", "what was discussed?"
model: haiku
allowed_tools:
  - Read
  - Grep
klabautermann-task-type: research
klabautermann-agent: researcher
---

But when someone asks "what skills do we have?", nobody wants to read YAML files. They want a list, a description, maybe some examples. The information exists - it just needs transformation.

The Generator

SkillDocsGenerator turns loaded skills into structured documentation:

class SkillDocsGenerator:
    """
    Generates documentation from skill definitions.

    Takes LoadedSkill objects and produces SkillDoc objects
    with extracted metadata, parameters, and trigger phrases.
    """

    def generate(self, skill: LoadedSkill) -> SkillDoc:
        """Generate documentation for a single skill."""

    def generate_all(self) -> dict[str, SkillDoc]:
        """Generate documentation for all loaded skills."""

    def generate_index(self, docs: dict[str, SkillDoc] | None = None) -> str:
        """Generate an index page listing all skills."""

Extracting What Matters

The generator doesn't just copy fields - it extracts semantic information:

Trigger Phrases

def _extract_trigger_phrases(self, description: str) -> list[str]:
    """Extract trigger phrases from description text."""
    patterns = [
        r'"([^"]+)"',                    # "phrase in quotes"
        r"'([^']+)'",                    # 'phrase in quotes'
        r"(?:say|ask|type)\s+([^,.]+)",  # say X, ask X
    ]
    phrases = []
    for pattern in patterns:
        matches = re.findall(pattern, description, re.IGNORECASE)
        phrases.extend(matches)
    return phrases

From "Use when user asks 'summarize this thread', 'TLDR', or 'what was discussed?'" it extracts: ["summarize this thread", "TLDR", "what was discussed?"]

Parameters

def _extract_parameters(self, skill: LoadedSkill) -> list[SkillParameter]:
    """Extract parameter documentation from payload schema."""
    schema = skill.klabautermann.payload_schema
    if not schema:
        return []

    params = []
    properties = schema.get("properties", {})
    required = set(schema.get("required", []))

    for name, prop in properties.items():
        params.append(SkillParameter(
            name=name,
            type=prop.get("type", "any"),
            required=name in required,
            default=prop.get("default"),
            description=prop.get("description"),
        ))
    return params

Payload schemas become documented parameters with types, defaults, and required markers.

Output Formats

Documentation can be rendered as markdown or HTML:

Markdown

def to_markdown(self) -> str:
    lines = [f"# {self.name}", "", self.description, ""]

    if self.trigger_phrases:
        lines.append("## Trigger Phrases")
        for phrase in self.trigger_phrases:
            lines.append(f'- "{phrase}"')
        lines.append("")

    if self.parameters:
        lines.append("## Parameters")
        lines.append("| Name | Type | Required | Default | Description |")
        lines.append("|------|------|----------|---------|-------------|")
        for p in self.parameters:
            lines.append(p.format_table_row())
        lines.append("")

    return "\n".join(lines)

HTML

def to_html(self) -> str:
    html = [f"
{self.name}
", f"
{self.description}
"]

    if self.trigger_phrases:
        html.append("
Trigger Phrases
    ")
        for phrase in self.trigger_phrases:
            html.append(f"
  • {phrase}
    • ")
        html.append("
")

    if self.parameters:
        html.append("
Parameters
")
        html.append("")
        html.append("")
        html.append("")
        html.append("")
        for p in self.parameters:
            html.append(p.format_html_row())
        html.append("
NameTypeRequiredDefaultDescription
")

    return "\n".join(html)

Index Generation

For skill catalogs, the generator creates an index page:

def generate_index(self, docs: dict[str, SkillDoc] | None = None) -> str:
    """Generate an index page listing all skills."""
    if docs is None:
        docs = self.generate_all()

    lines = ["# Skills Index", "", f"Found {len(docs)} skills.", ""]

    for name, doc in sorted(docs.items()):
        summary = doc.description.split("\n")[0][:100]
        lines.append(f"## [{name}]({name}.md)")
        lines.append(f"{summary}")
        lines.append("")

    return "\n".join(lines)

One command generates a navigable skills reference.

The Data Classes

@dataclass
class SkillParameter:
    """Documentation for a single skill parameter."""
    name: str
    type: str
    required: bool
    default: str | None
    description: str | None

    def format_table_row(self) -> str:
        required_str = "Yes" if self.required else "No"
        default_str = str(self.default) if self.default else "-"
        desc_str = self.description or "-"
        return f"| {self.name} | {self.type} | {required_str} | {default_str} | {desc_str} |"

@dataclass
class SkillDoc:
    """Generated documentation for a single skill."""
    name: str
    description: str
    model: str | None = None
    allowed_tools: list[str] = field(default_factory=list)
    user_invocable: bool = True
    task_type: str | None = None
    agent: str | None = None
    blocking: bool = True
    parameters: list[SkillParameter] = field(default_factory=list)
    trigger_phrases: list[str] = field(default_factory=list)
    body_sections: dict[str, str] = field(default_factory=dict)
    path: Path | None = None
