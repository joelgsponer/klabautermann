Mobile Shores

The CLI is home. The terminal is comfortable. But sometimes you're not at your desk.

Today, Klabautermann learned to speak Telegram.

The Setup

Three things make a Telegram bot:

  1. Talk to @BotFather on Telegram
  2. Create a bot, get a token
  3. Set TELEGRAM_BOT_TOKEN in your .env

That's it. Start Klabautermann with Telegram enabled and it begins polling for messages.

The Interface

Three commands:

  • /start - Welcome message with the nautical greeting
  • /help - What the bot can do
  • /status - Chat ID, user ID, voice status

Everything else is conversation. Same orchestrator, same knowledge graph, different port of entry.

Thread Isolation

Each Telegram chat gets its own thread:

CLI session: cli-abc123
Telegram chat: telegram-987654321

Ask "What did I talk about?" from CLI - you get CLI context. Ask from Telegram - you get Telegram context. No bleed. The spec demands it.

Voice Messages

Telegram voice messages work too:

  1. User sends voice note
  2. Download to temp file
  3. Transcribe with OpenAI Whisper
  4. Process as text
  5. Reply with transcription + response

Walking the dog, recording a voice note: "I just talked to Sarah about the Q3 budget."

Klabautermann captures it. The graph grows.

Authorization

Private bot? Set allowed_user_ids:

config:
  telegram:
    allowed_user_ids: [123456789, 987654321]

Empty list means public. Non-empty means whitelist only. Unauthorized users get a polite rejection.

The Architecture

TelegramDriver(BaseChannel)
├── start()         # Initialize, register handlers, start polling
├── stop()          # Graceful shutdown
├── send_message()  # Reply to chat
├── receive_message() # Forward to orchestrator
├── get_thread_id() # chat_id to thread UUID
├── _cmd_start()    # /start handler
├── _cmd_help()     # /help handler
├── _cmd_status()   # /status handler
├── _on_text()      # Text message handler
├── _on_voice()     # Voice message handler
└── _transcribe_audio() # Whisper integration

479 lines. 21 tests. Same patterns as CLI. Different transport.

The Code

async def _on_text(self, update: Update, context: ContextTypes.DEFAULT_TYPE) -> None:
    """Handle incoming text messages."""
    user_id = update.message.from_user.id

    # Authorization check
    if self.allowed_user_ids and user_id not in self.allowed_user_ids:
        await update.message.reply_text("Sorry, I don't recognize you.")
        return

    # Typing indicator
    await update.message.chat.send_action("typing")

    # Sanitize and process
    content = self._sanitizer.sanitize(update.message.text).sanitized
    thread_id = self.get_thread_id(update)
    response = await self.receive_message(thread_id=thread_id, content=content)

    await update.message.reply_text(response, parse_mode="Markdown")
