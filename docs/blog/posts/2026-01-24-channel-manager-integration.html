Hailing All Hands

The Officer of the Watch has always been vigilant. Deadlines approaching? Warning raised. Meeting in 15 minutes? Reminder sent. Task overdue? Error flagged. But until now, those alerts went... nowhere useful.

The Problem (#67)

The Officer generated alerts. The Channel Manager could broadcast to all channels. But they didn't talk to each other. Alerts were returned in API responses, but never actually sent anywhere.

If you weren't actively polling the Officer, you'd never know your deadline was approaching. Not very "proactive" for a proactive alert agent.

The Bridge Connection

Now the Officer has a direct line to the Channel Manager. When alerts are generated, they can be broadcast to every active channel - CLI, Telegram, whatever's connected.

officer = OfficerOfTheWatch(
    neo4j_client=client,
    channel_manager=get_channel_manager(),  # NEW
)

# Check conditions and broadcast alerts
result = await officer.check_conditions()
if result.alerts:
    send_result = await officer.send_alerts_to_channels(result.alerts)

Channel-Specific Formatting

Different channels need different formats. The Officer now speaks their language:

Telegram

⚠️ *Deadline*: Task 'Review PR' is due soon

Markdown format with emoji prefixes. Bold labels, clean readability.

CLI

⚠️ **Deadline**: Task 'Review PR' is due soon

Rich Markdown that renders beautifully in the terminal with emoji and bold text.

Priority Indicators

  • 🚨 CRITICAL - Something's on fire
  • ❌ ERROR - Something's already burned
  • ⚠️ WARNING - Smoke detected
  • ℹ️ INFO - Just keeping you informed

Graceful Failure Handling

Networks fail. Channels drop. The Officer handles it gracefully:

send_result = await officer.send_alerts_to_channels(alerts)

if send_result.all_delivered:
    print("All hands notified")
else:
    print(f"Delivered to: {send_result.channels_delivered}")
    print(f"Failed for: {send_result.channels_failed}")
    print(f"Errors: {send_result.errors}")

If Telegram is down but CLI is up, you still get your alerts on CLI. Partial success is still success.

The AlertSendResult

A new dataclass tracks exactly what happened:

@dataclass
class AlertSendResult:
    alerts_sent: int
    channels_delivered: list[str]
    channels_failed: list[str]
    errors: dict[str, str]  # channel_name -> error message

    @property
    def all_delivered(self) -> bool:
        return len(self.channels_failed) == 0

Message Protocol

You can also send alerts via the message protocol:

msg = AgentMessage(
    source_agent="orchestrator",
    target_agent="officer_of_the_watch",
    intent="officer_command",
    payload={
        "operation": "send_alerts",
        "alerts": [
            {
                "alert_type": "deadline_warning",
                "priority": "WARNING",
                "message": "Task 'Submit report' due in 4 hours",
            }
        ],
    },
    trace_id="trace-123",
)
