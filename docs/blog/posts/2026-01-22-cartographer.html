Mapping the Islands

As your knowledge graph grows, patterns emerge. Work contacts cluster with projects. Family members connect to home events. Hobbies link to related activities.

The Cartographer sees these patterns and gives them names.

The Problem with Flat Search

Search at the entity level works for specific queries. "What's Sarah's email?" "When's my meeting?" But some questions are broader:

  • "What's going on at work?"
  • "Who are my closest friends?"
  • "What hobbies have I been neglecting?"

These need a higher-level view. Not individual nodes, but clusters of meaning.

Knowledge Islands

The Cartographer uses Neo4j's Graph Data Science library to run community detection:

from klabautermann.agents import Cartographer, CartographerConfig

config = CartographerConfig(
    projection_name="klabautermann-community",
    min_community_size=3,
)

cartographer = Cartographer(neo4j_client=neo4j, config=config)

# Run detection
result = await cartographer.detect_communities()

print(f"Created: {result.communities_created}")
print(f"Updated: {result.communities_updated}")
print(f"Members assigned: {result.total_members_assigned}")

The Louvain Algorithm

At its heart, the Cartographer uses Louvain community detection. The algorithm:

  1. Projects the graph into memory (nodes and undirected relationships)
  2. Optimizes modularity by grouping densely connected nodes
  3. Returns community assignments for each node
// GDS projection
CALL gds.graph.project(
    'klabautermann-community',
    ['Person', 'Organization', 'Project', 'Hobby', 'Note'],
    {
        WORKS_AT: {orientation: 'UNDIRECTED'},
        KNOWS: {orientation: 'UNDIRECTED'},
        FRIEND_OF: {orientation: 'UNDIRECTED'},
        FAMILY_OF: {orientation: 'UNDIRECTED'},
        CONTRIBUTES_TO: {orientation: 'UNDIRECTED'}
    }
)

// Louvain detection
CALL gds.louvain.stream('klabautermann-community')
YIELD nodeId, communityId

Theme Classification

Raw community IDs aren't helpful. The Cartographer classifies each cluster by its dominant theme:

from klabautermann.agents import CommunityTheme, classify_theme

labels = ["Organization", "Project", "Task", "Person"]
theme = classify_theme(labels)

# CommunityTheme.PROFESSIONAL

Themes are weighted by node label:

  • Professional: Organization (1.0), Project (0.8), Task (0.7)
  • Hobbies: Hobby (1.0)
  • Health: HealthMetric (1.0), Routine (0.5)
  • Family/Social: Person (0.3 each)

Community Nodes

Detected communities become first-class graph citizens:

(c:Community {
    uuid: "comm-123",
    name: "Professional Island",
    theme: "professional",
    node_count: 15,
    detected_at: timestamp(),
    created_at: timestamp()
})

// Members link via PART_OF_ISLAND
(person:Person)-[:PART_OF_ISLAND {weight: 1.0}]->(c)
(org:Organization)-[:PART_OF_ISLAND {weight: 1.0}]->(c)

Updating Existing Islands

Communities evolve. New nodes join, old connections fade. The Cartographer handles this with overlap detection:

# Find existing community with 50% member overlap
existing = await cartographer._find_existing_community(
    theme=CommunityTheme.PROFESSIONAL,
    member_uuids=new_members
)

if existing:
    # Update existing community
    await cartographer._update_community(existing, new_members)
else:
    # Create new community
    await cartographer._create_community(theme, new_members)

Querying Islands

Once mapped, islands are queryable:

# Get all communities
communities = await cartographer.get_communities()

# Filter by theme
professional = await cartographer.get_communities(
    theme=CommunityTheme.PROFESSIONAL
)

# Get island members
members = await cartographer.get_community_members("comm-123")

# Statistics
stats = await cartographer.get_community_statistics()
# {
#     "total_communities": 5,
#     "avg_members": 12.4,
#     "by_theme": {"professional": 2, "social": 2, "hobbies": 1}
# }

Multi-Level Search

With Knowledge Islands in place, the Researcher can adapt its zoom level:

  • Macro: "What's happening in my professional life?" - Search Community nodes
  • Meso: "What's the status of Project X?" - Search Project/Note nodes
  • Micro: "What's John's email?" - Search entity facts

The query intent determines the zoom. Islands enable the macro view that was previously impossible.
