Smooth Sailing on Telegram

Telegram's Markdown parser is strict. Special characters can break it. A response containing [0] or 2+2=4 would fail with "Can't parse entities."

Not anymore.

The Problem

Telegram supports Markdown formatting for messages. But its parser treats certain characters as formatting markers:

  • [ ] - link text markers
  • ( ) - link URL markers
  • * - bold markers
  • _ - italic markers
  • ` - code markers
  • . + = > # - and more

An unescaped bracket or mismatched asterisk causes the whole message to fail.

The Solution

Escape special characters before sending:

from klabautermann.channels.markdown_formatter import escape_markdown

text = "Array [0] + Array [1] = Result"
safe = escape_markdown(text)
# "Array \\[0\\] \\+ Array \\[1\\] \\= Result"

Preserving Intentional Formatting

Not all special characters should be escaped. If the response contains intentional formatting:

*bold* and _italic_ and `code`

We want to keep that. The formatter detects formatting patterns and protects them:

text = "*Bold* text with [array] value"
safe = escape_markdown(text)
# "*Bold* text with \\[array\\] value"

Bold stays bold. Brackets get escaped.

Graceful Fallback

Sometimes formatting still fails - malformed responses, edge cases. Instead of crashing:

async def _safe_reply(self, message, text):
    try:
        await message.reply_text(formatted, parse_mode="Markdown")
    except Exception as e:
        if "parse" in str(e).lower():
            # Markdown failed, send plain text
            await message.reply_text(text)
        else:
            raise

Users see the message either way. Formatting is nice-to-have, not must-have.

What Gets Escaped

[ ] ( ) ~ ` > # + - = | { } . !

Each becomes \[, \], etc.

What Gets Preserved

  • *bold text*
  • _italic text_
  • `inline code`
  • ```code blocks```
  • [link text](url)
