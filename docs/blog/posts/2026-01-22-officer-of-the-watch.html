Standing Watch

On any ship, someone has to keep watch. Someone has to scan the horizon for danger, check the schedule, and wake the Captain before the storm hits.

Meet the Officer of the Watch.

The Problem

Knowledge graphs are great at answering questions. But they're reactive. They wait for you to ask.

Sometimes you need to be reminded. That task is due tomorrow. That meeting starts in 15 minutes. That deadline you set last week? It passed yesterday.

Deadline Warnings

The Officer watches your tasks. When one gets close to its due date, you'll know:

from klabautermann.agents import OfficerOfTheWatch, OfficerConfig

config = OfficerConfig(
    deadline_warning_hours=24,  # Warn 24 hours before due
    alert_debounce_minutes=60,  # Don't repeat within 1 hour
)

officer = OfficerOfTheWatch(neo4j_client=neo4j, config=config)

# Find tasks due soon
alerts = await officer.check_upcoming_deadlines()

for alert in alerts:
    print(f"{alert.priority}: {alert.message}")
    # WARNING: Task due in 12.5h: Review PR (Backend)

High-priority tasks get WARNING alerts. The rest get INFO. Overdue tasks? Those are ERROR level.

Meeting Reminders

Fifteen minutes before a meeting, the Officer speaks up:

# Find meetings starting soon
alerts = await officer.check_upcoming_meetings()

for alert in alerts:
    print(alert.message)
    # Meeting in 10min: Team Standup at Zoom

Location included, if your calendar has it. Time until the meeting, so you know exactly how long you have.

Quiet Mode

Focus time is sacred. When the Officer sees a "Deep Work" block on your calendar, it stays quiet:

is_deep_work = await officer.is_deep_work_time()
# True if you're in a "Deep Work", "Focus", or "Do Not Disturb" event

# Only CRITICAL alerts break the silence
if officer._quiet_mode and alert.priority != AlertPriority.CRITICAL:
    return  # Stay quiet

The Officer looks for calendar events with keywords like "Deep Work", "Focus", or "Do Not Disturb". While you're in one, only critical alerts get through.

Alert Debouncing

Nobody wants the same reminder every minute. The Officer remembers what it already told you:

config = OfficerConfig(
    alert_debounce_minutes=60,  # Wait 1 hour before repeating
)

# Same alert for same entity won't fire again for an hour

Full Monitoring

Run all checks at once:

result = await officer.check_conditions()

print(f"Found {result.alerts_found} alerts")
print(f"Sending {result.alerts_sent} (after filtering)")
print(f"Quiet mode: {result.quiet_mode}")

for alert in result.alerts:
    print(f"  [{alert.priority}] {alert.message}")

Inter-Agent Communication

Like all Klabautermann agents, the Officer responds to messages:

message = AgentMessage(
    source_agent="orchestrator",
    target_agent="officer_of_the_watch",
    intent="maintenance",
    payload={"operation": "check_conditions"}
)

response = await officer.process_message(message)
# response.payload contains AlertCheckResult data
