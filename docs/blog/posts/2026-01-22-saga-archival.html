Archiving Tales: Saga Summaries

The Bard tells stories. Sagas span chapters. Eventually, every tale must end.

But endings should be more than just silence. When a saga concludes, it deserves a summary - a record of its journey.

The Problem

When sagas completed (5 chapters) or timed out (30 days), they simply stopped. No summary. No way to remember what the tale was about without reading every chapter.

LORE_SYSTEM.md Section 3.3 specified the solution: archive with summaries.

The Solution (#125, #126)

Two issues. One cohesive feature:

  • #125: Generate summary when archiving
  • #126: Create Note node with SUMMARIZES relationships

How It Works

When archive_saga is called:

  1. Gather all chapters of the saga
  2. Generate a summary capturing the narrative arc
  3. Create a Note node to store the summary
  4. Link the Note to all episodes with SUMMARIZES
  5. Mark all episodes as archived

The Graph Schema

After archival, the graph looks like this:


         ┌──────────────────────────────────────┐
         │  Note                                │
         │  title: "Saga: The Lost Password"    │
         │  content: "Summary of the tale..."   │
         │  source: 'lore_archive'              │
         └──────────────────────────────────────┘
                           │
                           │ [:SUMMARIZES]
         ┌─────────────────┼─────────────────┐
         ↓                 ↓                 ↓
    ┌─────────┐      ┌─────────┐      ┌─────────┐
    │ Episode │      │ Episode │      │ Episode │
    │ Ch 1    │      │ Ch 2    │      │ Ch 3    │
    │ archived│      │ archived│      │ archived│
    └─────────┘      └─────────┘      └─────────┘

Summary Generation

The summary captures beginning and end:

async def _generate_saga_summary(
    self,
    episodes: list[LoreEpisode],
    trace_id: str | None = None,
) -> str:
    """Generate a summary of a saga from its episodes."""
    if not episodes:
        return "An untold tale, lost to the digital mists."

    saga_name = episodes[0].saga_name

    first_chapter = episodes[0].content[:100] + "..."
    last_chapter = episodes[-1].content[:100] + "..."

    summary = (
        f"The tale of '{saga_name}' spans {len(episodes)} chapters, "
        f'beginning with: "{first_chapter}" '
        f'and concluding with: "{last_chapter}"'
    )

    return summary

Simple for now. A future implementation could use LLM to generate richer summaries.

The Archive Note

Creating the Note with relationships:

query = """
CREATE (n:Note {
    uuid: $note_uuid,
    title: $title,
    content: $summary,
    source: 'lore_archive',
    created_at: $now_ms
})
WITH n
MATCH (le:LoreEpisode {saga_id: $saga_id})
SET le.archived = true
WITH n, collect(le) as episodes
UNWIND episodes as ep
CREATE (n)-[:SUMMARIZES {created_at: $now_ms}]->(ep)
RETURN n.uuid as uuid
"""

One query does three things:

  1. Create the Note
  2. Mark episodes as archived
  3. Create all SUMMARIZES relationships

Process Message Handlers

Two new operations:

elif operation == "archive_saga":
    saga_id_param = payload.get("saga_id", "")
    archive_result = await self.archive_saga(
        saga_id=saga_id_param, trace_id=trace_id
    )
    result_payload = {"archived": True, **archive_result}

elif operation == "get_archived_saga":
    saga_id_param = payload.get("saga_id", "")
    archived_saga = await self.get_archived_saga(
        saga_id=saga_id_param, trace_id=trace_id
    )
    result_payload = {"saga": archived_saga, "found": archived_saga is not None}

The Result

Tales now have proper endings. When a saga completes - whether through reaching chapter 5 or timing out after 30 days - the Bard creates a summary and stores it in a Note.

The SUMMARIZES relationship links the summary back to all the original episodes. The graph becomes a library of tales, each archived with a note about what it contained.

Every tale told is a tale remembered. Every saga archived is a story preserved.
