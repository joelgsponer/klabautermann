Saving the Graph

The shell scripts exist. They use neo4j-admin dump at the Docker level. But what about Python-level exports? What about JSON you can inspect, version, and move between environments?

Now you can.

The Backup

Create a snapshot of your entire knowledge graph:

from klabautermann.memory import create_backup, save_backup_to_file

# Create snapshot
snapshot = await create_backup(neo4j_client)

# Check what we got
print(f"Nodes: {snapshot.metadata.node_count}")
print(f"Relationships: {snapshot.metadata.relationship_count}")
print(f"Labels: {snapshot.metadata.node_labels}")

# Save to file
await save_backup_to_file(snapshot, "backups/knowledge-2026-01-22.json")

The snapshot captures everything: all nodes with their labels and properties, all relationships with their types and endpoints, plus metadata about when it was created and what version format it uses.

The Restore

Bring your data back:

from klabautermann.memory import load_backup_from_file, restore_backup

# Load snapshot
snapshot = await load_backup_from_file("backups/knowledge-2026-01-22.json")

# Restore to database
result = await restore_backup(
    neo4j_client,
    snapshot,
    clear_existing=True,  # Start fresh
)

print(f"Restored {result.nodes_restored} nodes")
print(f"Restored {result.relationships_restored} relationships")

if result.errors:
    print(f"Errors: {result.errors}")

The clear_existing option removes all existing data first. Useful for migrations, test environments, or disaster recovery.

Validation

Before you restore, validate:

from klabautermann.memory import validate_backup

errors = await validate_backup(snapshot)

if errors:
    for error in errors:
        print(f"Problem: {error}")
else:
    print("Backup is consistent")

Validation checks:

  • Node count matches metadata
  • Relationship count matches metadata
  • All relationships reference existing nodes
  • No duplicate UUIDs within the same label

The JSON Format

Readable, portable, versionable:

{
  "metadata": {
    "created_at": "2026-01-22T14:30:00+00:00",
    "version": "1.0",
    "node_count": 247,
    "relationship_count": 312,
    "node_labels": ["Person", "Organization", "Project"],
    "relationship_types": ["WORKS_AT", "KNOWS", "CONTRIBUTES_TO"]
  },
  "nodes": [
    {
      "element_id": "4:abc:0",
      "labels": ["Person"],
      "properties": {
        "uuid": "person-123",
        "name": "John Doe",
        "email": "john@example.com"
      }
    }
  ],
  "relationships": [
    {
      "type": "WORKS_AT",
      "from_uuid": "person-123",
      "to_uuid": "org-456",
      "properties": {"title": "Engineer"}
    }
  ]
}

Clear Database

When you need a clean slate:

from klabautermann.memory import clear_database

deleted = await clear_database(neo4j_client)
print(f"Deleted {deleted} nodes")

Deletes in batches of 1000 to avoid memory issues. Relationships are removed automatically when their nodes go.

Use Cases

  • Disaster Recovery: Regular JSON backups alongside neo4j-admin dumps
  • Environment Migration: Export from dev, import to staging
  • Testing: Snapshot known state, run tests, restore
  • Debugging: Inspect the JSON to understand graph structure
  • Version Control: Track knowledge graph changes over time
