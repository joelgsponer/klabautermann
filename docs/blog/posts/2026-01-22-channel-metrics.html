Observing the Channels

You can't improve what you can't measure. Klabautermann already has Prometheus metrics for agents, APIs, graphs, and LLMs. Today: channels.

The Gap

Channels handle user messages. CLI, Telegram, soon Discord. How do you know they're healthy? How fast are responses? Which channel gets the most traffic?

Before today: logs. Check the logs. Grep through timestamps. Hope nothing broke while you weren't watching.

Eight New Metrics

Now there's a better way:

klabautermann_channel_messages_total{channel_name="telegram"}
klabautermann_channel_response_latency_ms{channel_name="cli"}
klabautermann_channel_errors_total{channel_name="telegram",error_type="connection"}
klabautermann_channel_status{channel_name="cli"}
klabautermann_channel_healthy{channel_name="telegram"}
klabautermann_channel_broadcasts_total
klabautermann_channel_broadcast_deliveries_total{channel_name="cli",status="success"}
klabautermann_channel_active_count

Counters, gauges, histograms. The standard Prometheus primitives.

What They Track

  • Messages: Total count per channel. See which channel gets used most.
  • Latency: Response time histogram. P50, P95, P99 out of the box.
  • Errors: Count by channel and error type. Spot failing channels fast.
  • Status: Running (1) or stopped (0). Alert when channels go down.
  • Health: Healthy (1) or unhealthy (0). Track auto-restart events.
  • Broadcasts: Total broadcast messages sent.
  • Deliveries: Success/failure per channel. Know if broadcasts reach everyone.
  • Active Count: How many channels are currently running.

Integration

The ChannelManager now records metrics automatically:

async def _start_channel(self, name: str, channel: BaseChannel) -> None:
    await channel.start()
    self._status[name] = ChannelStatus.RUNNING
    set_channel_status(name, running=True)
    set_channel_active_count(len(self.active_channels))

Start a channel: metric updated. Stop a channel: metric updated. Health check runs: metric updated. Broadcast sends: metric updated.

Enhanced Message Recording

The record_message() method now accepts latency:

def record_message(self, channel_name: str, latency_ms: float | None = None) -> None:
    if channel_name in self._message_counts:
        self._message_counts[channel_name] += 1
        record_channel_message(channel_name)
        if latency_ms is not None:
            record_channel_latency(channel_name, latency_ms)

Pass the response time, get histogram data. Don't pass it, still get the count.

Error Tracking

Errors are categorized by type:

record_channel_error("telegram", "connection")
record_channel_error("telegram", "timeout")
record_channel_error("cli", "restart_failure")

Filter by error type in Grafana. Build alerts for specific failure modes.

Broadcast Observability

Every broadcast now leaves a trace:

async def broadcast(self, content: str, ...) -> BroadcastResult:
    record_channel_broadcast()

    for name in self.active_channels:
        try:
            await channel.send_message(...)
            record_channel_broadcast_delivery(name, success=True)
        except Exception:
            record_channel_broadcast_delivery(name, success=False)
            record_channel_error(name, "broadcast_failure")

Know exactly how many broadcasts you've sent. Know exactly which channels received them.

Usage

The metrics export via the existing /metrics endpoint:

curl http://localhost:8000/metrics | grep channel

Point Prometheus at it. Build Grafana dashboards. Set up alerts.
