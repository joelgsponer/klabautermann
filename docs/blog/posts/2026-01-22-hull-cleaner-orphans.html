Orphans in the Graph

Weak relationships were just the beginning. The Hull Cleaner now hunts orphans.

The Problem

Knowledge graphs aren't just about relationships. They're about nodes too. And sometimes, through edge cases in ingestion or bugs in thread management, you end up with messages that belong to no thread.

They sit there. Unconnected. Unreachable. Taking up space.

Finding Orphans

The Hull Cleaner integrates with existing orphan detection infrastructure:

orphans = await cleaner.find_orphan_messages()

for orphan in orphans:
    print(f"Orphan: {orphan.uuid}")
    print(f"  Role: {orphan.role}")
    print(f"  Content: {orphan.content[:50]}...")

It delegates to orphan_cleanup.py under the hood, keeping all graph analysis in one place.

Removing Orphans

Same pattern as weak relationships. Dry-run first:

# Preview what would be removed
result = await cleaner.remove_orphan_messages(dry_run=True)

print(f"Found {result.nodes_found} orphan messages")
print(f"Would remove {result.nodes_removed}")

for entry in result.audit_entries:
    print(f"  {entry.action}: {entry.reason}")

When you're ready:

# Actually remove orphans
result = await cleaner.remove_orphan_messages(dry_run=False)

print(f"Removed {result.nodes_removed} orphan messages")

Full Maintenance Now Includes Orphans

scrape_barnacles() now handles both weak relationships and orphan messages:

result = await cleaner.scrape_barnacles(dry_run=True)

# Combined stats from both operations
print(f"Relationships to prune: {result.relationships_pruned}")
print(f"Orphan nodes to remove: {result.nodes_removed}")
print(f"Total audit entries: {len(result.audit_entries)}")

Configuration

New options in HullCleanerConfig:

config = HullCleanerConfig(
    # Existing weak relationship settings
    prune_weak_relationships=True,
    weak_relationship_threshold=0.2,
    weak_relationship_age_days=90,

    # New orphan settings
    remove_orphan_messages=True,  # Enable/disable
    orphan_batch_size=50,         # Batch size for deletion

    # Safety limits apply to both
    max_deletions_per_run=1000,
    dry_run_by_default=True,
)

Inter-Agent Communication

Other agents can request orphan operations through messages:

# Find orphans
message = AgentMessage(
    source_agent="orchestrator",
    target_agent="hull_cleaner",
    intent="maintenance",
    payload={"operation": "find_orphan_messages"}
)

# Remove orphans (dry-run)
message = AgentMessage(
    source_agent="orchestrator",
    target_agent="hull_cleaner",
    intent="maintenance",
    payload={"operation": "remove_orphan_messages", "dry_run": True}
)
